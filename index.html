Reusable elements

Element semantics

Sidebar 承担全局导航和工作区切换，保持轻量、固定、可折叠。
Page header 负责页面定位、状态反馈和模式切换，不堆过多按钮。
Accent button 只给当前页面最主要的动作，避免多个同权重主按钮同时出现。
Status pill 用颜色表达流程状态，不承担复杂说明；文案必须短。
Metric card 展示单一关键指标，一张卡只表达一个结论。
Dark detail panel 用于承载选中对象的深层信息，作为页面的次级聚焦区。
Inline edit shell 用于表格原位修改，减少弹窗，保持上下文连续。
Confirm dialog 仅用于删除、覆盖等高风险动作，视觉上明确风险等级。
Engineering rules

Persistence constraints

01
Single-instance JSON architecture

适用于内部团队、低并发写入、中等数据量，不以数据库替代为目标。

02
Atomic writes

每次写入都要先校验，再归档旧文件，写临时文件，最后 rename 覆盖。

03
Serialized writes

同一 dataset 的写入必须串行，避免单进程内的部分覆盖竞争。

04
Startup validation

服务启动时确保数据文件存在并完成校验，发现坏数据直接 fail fast。
